# Daily/Periodic Health Analysis — 让 AI 小结「分析到点子上」(Insightful, Not Generic)

**Status:** Research + design findings, not medical advice.
**Date:** 2026-07-24
**Scope:** 个人自用高血压自追踪 App 的 AI「近期健康小结」(daily/periodic)。当前实现见 `src/server/ai.ts`；本文件给出从「星座运势式套话」升级为「像心内科医生一样有针对性」的依据与落地方案。

> **医学免责声明:** 下述阈值汇编自已发表的临床指南，**仅供工程/产品规格参考，不构成医疗建议**，不得用于个体诊断或治疗决策。上线面向真实多用户前须心内科/高血压专科医生签字确认。AI 输出框定为「健康提醒而非诊断」，不得出具用药/剂量调整建议。

---

## 0. 先诊断当前 AI 为什么「假大空」（代码现状）

当前 `src/server/ai.ts` 的实现决定了输出必然泛泛：

| 维度 | 现状 (`ai.ts`) | 后果 |
|---|---|---|
| 数据范围 | 仅近 **7 次**读数 | 样本太小，看不出趋势/变异性/达标率 |
| 上下文内容 | 只有原始读数行 + 平均血压 + 达标率 | 无用药依从性、无清晨聚合、无症状关联、无档案/合并症 |
| 系统 prompt | 「整体情况/达标情况/需要关注的点」三段套话 | 模型只会套模板 |
| 输出约束 | 「150 字以内」 | 字数太少无法承载具体发现 |
| 数据组织 | 原始文本拼接，未做特征提取 | 模型自己算不出「连续3天清晨≥135」「漏服当天偏高」这类交叉信号 |

**结论:** 做到「到点子上」的杠杆点是 **上下文装配（context assembly）**——把 App 已有但分散的信号（清晨 vs 晚间、逐日波动、用药打卡、症状、时段）**预先交叉计算成结构化特征**，再喂给模型。模型擅长「按证据组织语言」，不擅长「在一堆原始行里自己发现关联」。这与临床 LLM 文献一致（见 §3.1）。

> 注：`src/server/report.ts` 已经为「医生报告」装配了 30 天读数+档案+用药+统计+偏高峰值。AI 小结应复用同一条装配线，并在此基础上增加清晨聚合、依从性、症状、趋势方向等维度。

## 1. 临床上值得呈现的高血压模式（阈值 + 为什么重要 + 可执行 nudge）

> 核心区分（务必先读）：**清晨高血压 ≠ 晨峰**。家庭自测 App 能可靠识别**清晨高血压**，但**不能**测真正的**晨峰(MBPS)**——后者需要 24h 动态血压(ABPM)对比夜间谷值与晨起峰值。App 里只能呈现「清晨血压偏高」，并提示「真晨峰需 ABPM 确诊」，不可越界宣称测到了晨峰。

### 1.1 清晨高血压 (morning hypertension) — App 能检测

- **标准:** 晨起家庭血压 **≥ 135/85 mmHg**（家庭血压标准）。来源：《中国高血压防治指南(2024 修订版)》诊断阈值表（诊室≥140/90、**家庭≥135/85**、24h≥130/80、白天≥135/85、夜间≥120/70）http://www.chl-bha.org.cn/Public/ueditor/php/upload/20240815/17236841017752.pdf ；《2019 中国家庭血压监测指南》 https://file.medtrib.cn/pdf/8a0522ac-ef54-4d24-afa3-0e51edbd70b2/2019%20中国家庭血压监测指南.pdf
- **为什么重要:** 清晨(晨起~上午10点)是心脑血管事件(卒中、心梗、猝死)最高发的时段；清晨高血压是卒中的独立预测因子，亚洲人群尤为显著（HOPE Asia 综述 Sogunuru/Kario 2019, *J Clin Hypertens*）https://pmc.ncbi.nlm.nih.gov/articles/PMC8030409/
- **可执行 nudge（非用药）:** 「近5个清晨均值142/88，连续3天≥135/85，晚间却全部达标——这是清晨高血压的典型表现，也是全天最高危时段。建议晨起1小时内、服药前、早餐前测量；把这几天清晨记录带给医生评估清晨时段的降压覆盖。**注：真『晨峰』幅度需24h动态血压确诊。**」

### 1.2 晨峰 (morning surge, MBPS) — App 不能直接测，仅作提示

- **标准(ABPM):** sleep-trough MBPS = (醒后2h均值) − (夜间最低血压1h均值)；**≥35 mmHg** 为升高，**≥55 mmHg**(Kario 最高十分位)为高危。pre-awakening MBPS = (醒后2h) − (醒前2h)。来源：HOPE Asia Network 定义 https://pmc.ncbi.nlm.nih.gov/articles/PMC8030409/
- **预后证据:** Kario 2003 (JMS-ABPM, *Circulation*): 晨峰≥55 mmHg 组卒中发生率 19.0% vs 7.3%，**RR 2.7**，独立于24h血压 https://pubmed.ncbi.nlm.nih.gov/12642361/ ；IDACO(Li 2010) 顶层十分位>37 mmHg: CV 事件 HR 1.30 https://pmc.ncbi.nlm.nih.gov/articles/PMC8030409/ (Table 4)
- **App 立场:** 没有夜间读数，**不要从家庭读数宣称「晨峰」**。只能提示「清晨血压偏高，若持续建议做24h动态血压评估晨峰」。

### 1.3 血压变异性 (day-to-day / visit-to-visit variability, BPV)

- **标准:** 无单一通用阈值；临床多作连续量分析(SD/CV/VIM/ARV)。家庭 App 实用指标 = 晨起收缩压的**日间变异系数(CV%)**；SD **≥15 mmHg** 被佐证为日间收缩压变异病理升高的切点（正常上限 95%CI≈14.9）。来源：Ma 2021 (PMC7943313, 单次访内 BPV 与靶器官损害) https://pmc.ncbi.nlm.nih.gov/articles/PMC7943313/ ；顶层meta见 Rothwell 2010(*Lancet*) 访间变异独立预测卒中。
- **为什么重要:** Heshmatollah 2022 (Rotterdam, *PLOS Med*, 9958 人, 10.1y): SBP 变异每+1SD → 任何卒中 HR 1.11、**出血性卒中 HR 1.27**；长期滞后分析任意卒中 HR 1.29–1.47。**上升和下降都预测卒中——是『波动』本身伤血管，不是方向。** https://journals.plos.org/plosmedicine/article?id=10.1371/journal.pmed.1003942
- **机制:** 反复血压波动→动脉壁机械应力→动脉硬化、内皮功能紊乱→脑小血管病/白质病变。
- **可执行 nudge:** 「本周晨起收缩压在122–151之间跳动(CV≈12%)。『稳定的130』比『波动的130』更安全——波动本身独立增加卒中风险。常见诱因：漏服/晚服、高盐日、睡眠差、饮酒。结合您的用药日志可定位诱因。」

### 1.4 达标率 / 控制率 (control rate)

- **达标定义(家庭):** 平均 **< 135/85 mmHg**（收缩≤135 且 舒张≤85，AND 逻辑）。这与本仓 `src/lib/bp.ts` 的 `isControlled` 一致。
- **个人「控制良好」参考线:** 监测日里 **≥75–80%** 达标为「well-controlled」（与依从性 PDC 80% 惯例对齐，见 §1.5）。
- **为什么控制重要:** 《国家基层高血压防治管理指南2025》: SBP 每降 **10 mmHg**(或 DBP 5 mmHg) → 全因死亡 −10–15%、**卒中 −35%**、冠心病下降。这是「为什么达标值得努力」最强的单条证据。 https://bookcafe.yuntsg.com/ueditor/jsp/upload/file/20251003/1759482019584069121.pdf
- **中国流行病学(《2024指南》):** 加权患病率 27.5%；知晓率 51.6%、治疗率 45.8%、**控制率仅 16.8%**。
- **可执行 nudge:** 报告个人达标率并对标自身趋势：「本周达标 5/7 天(71%)，较上周43%回升。您晚间全部达标，清晨是达标漏防的时段——把清晨管住，达标率就上去了。」（对标用户**自己的**趋势，而非泛泛阈值。）

### 1.5 用药依从性 ↔ 血压 (adherence ↔ BP)

- **标准:** **PDC(覆盖天数比) ≥ 80%** 为「完全依从」，是临床结局与质量度量的通用阈值。来源：AHA 科学声明 (Choudhry 2021, *Hypertension*, 415 引) https://pmc.ncbi.nlm.nih.gov/articles/PMC11485247/
- **关键证据:** 漏服是「表观难治性高血压」的主要驱动；短信提醒(Bobrow/StAR) → BP −2 mmHg、依从性 +5%；药师辅导+板装药(Lee 2006) → SBP −6.9 mmHg、依从性 69%→96%。半衰期长的药对漏服更「宽容」。
- **关键告诫(不要归咎患者):** AHA 强调**临床惰性(医生未及时强化治疗)**常比患者漏服贡献更大。nudge 应「提出问题」而非「指责」。
- **可执行 nudge:** 「本周7天打卡5天(71%，略低于80%良好依从线)。您漏服的两个清晨(周三/六)当天晚间血压均值148/92，比按时服的日子(132/84)高约16/8 mmHg——时间上相关。是什么打断了打卡(忘记/时间/副作用)?下次复诊和医生聊聊，也许有更省心的方案。」

### 1.6 症状 ↔ 血压关联 —— 关键的「症状不靠谱」认知

- **证据:** WHO:「大多数高血压患者**没有任何症状**，测压是唯一知道的办法」 https://www.who.int/zh/news-room/fact-sheets/detail/hypertension 。Mayo: 症状(气短/视物模糊/头痛)只在**极少数、极高水平**时出现 https://www.mayoclinic.org/zh-hans/diseases-conditions/high-blood-pressure/symptoms-causes/syc-20373410 。中国中青年高血压常**隐性/无症状**(~70%不自知)。《中国高血压患者教育指南》列出提示症状(头晕头痛/眼花耳鸣/心悸胸闷/乏力/夜尿增多)但强调**不能据症状判断血压水平** https://www.nccd.org.cn/UploadFile/201504/20150418172843860860.pdf
- **App 能呈现的「到点」洞察 = 症状与血压的三种错配:**
  1. **症状有 + 血压正常** → 症状多半与血压无关（焦虑/其他），别误读；
  2. **无症状 + 血压高** → 危险的「沉默」型，正是家庭监测的价值所在；
  3. **症状有 + 血压高 + 同时段反复** → 时间上相关，值得记给医生（如头痛总聚在6–10点晨峰窗）。
- **可执行 nudge(沉默型):** 「本周4次读数≥135/85，但您没记录任何头晕/头痛。这是高血压最该警惕的状态——『没感觉≠血压正常』。请继续记录，复诊带上数据。」

### 1.7 趋势方向 + 未达标/隐性模式 (trend / masked / uncontrolled)

- **趋势方向:** 用「近7天 vs 前7天(或 vs 近30天)」的均值/达标率对比表达「改善/恶化」。每降 10 mmHg 收缩 ≈ 卒中风险 −35%（§1.4），可用来量化鼓励。
- **隐性(masked)/未控制(uncontrolled)模式:**
  - *Masked:* 诊室正常但家庭/动态升高——家庭监测就是为抓它。若用户某段时间家庭读数偏高但「自我感觉良好」，正是 masked。
  - *Uncontrolled:* 达标率长期 <50% 或均值持续 ≥140/90(家庭)→ 提示当前方案可能不足，**建议复诊评估**（非自行调药）。
- **危象红线:** 任一读数 **≥180/110** → 明确提醒「尽快就医」，不要自行加药。与本仓 `getBpStatus` 的 `crisis` 一致。

### 模式速查表

| 模式 | 标准/阈值 | 为何重要 | App 可呈现 |
|---|---|---|---|
| 清晨高血压 | 晨起家庭 BP ≥135/85 | 卒中最高发时段；独立预测因子 | ✅ 直接 |
| 晨峰 MBPS | ABPM sleep-trough ≥35/>=55mmHg | ≥55→卒中 RR 2.7 | ❌ 仅提示需 ABPM |
| 逐日变异 | 晨起 SBP CV% 偏高 / SD≥15 | 波动本身独立预测卒中 | ✅ 计算 CV |
| 达标率 | 家庭读数 ≤135/85 占比 | SBP −10→卒中 −35% | ✅ 直接 |
| 依从性 | PDC ≥80% 良好 | 漏服↔BP↑；表观难治性主因 | ✅ 由 med_log 计算 |
| 症状↔BP | 三种错配 | 无症状≠正常；沉默型最险 | ✅ 交叉分析 |
| 趋势 | 近7天 vs 前7天均值/达标率 | 量化改善/恶化 | ✅ 计算 |
| 危象 | ≥180/110 | 急诊风险 | ✅ 红线 |

## 2. 「到点子上」vs「星座运势」——检查清单 + 示例

### 2.1 一份好的每日/周期小结应击中的维度（检查清单）

每份小结**至少**落地以下 5–6 项中适用的那些（没有数据时如实说「数据不足」，不要硬编）：

- [ ] **A. 达标率 + 同比:** 「本周达标率 X%(n/N)，较上周 Y% ↑/↓」——带数字与方向。
- [ ] **B. 清晨维度:** 「近 K 个清晨均值 S/D，连续 N 天 ≥135/85」——抓清晨高血压；说明晚间是否正常。
- [ ] **C. 依从性 ↔ BP 相关:** 「依从率 X%；漏服的当天血压较按时日高约 Δ/Δ」——把打卡与血压**同时间轴**对齐。
- [ ] **D. 症状 ↔ BP 相关:** 指出三种错配之一（沉默型最值得点）。
- [ ] **E. 趋势方向:** 「近7天 vs 前7天，改善/恶化 Δ mmHg」，并用 SBP −10≈卒中 −35% 给鼓励/警醒。
- [ ] **F. 波动性:** 「CV≈X%，血压在 a–b 间跳动」+ 一句诱因提示。
- [ ] **G. 红线/边界:** ≥180/110 → 就医；连续多日 uncontrolled → 建议复诊。
- [ ] **H. 可执行的下一步:** 一条**具体的、非用药的**行为建议（测量时机、记录习惯、带数据复诊、与医生聊依从性障碍）。
- [ ] **I. 诚实标注不确定性:** 数据不足/测量不规范时明说（如「仅2天数据，趋势不具结论性」）；「晨峰需 ABPM」。

> 反模式(星座运势): 「您的血压整体偏高，请注意饮食、规律作息、按时服药、保持心情舒畅，如有不适请就医。」——零数据、零交叉、放之四海皆准。这是当前 `ai.ts` 150字+套话 prompt 必然产出。

### 2.2 示例分析（中文，读起来像有心的医生，非算命）

> 例 1 · 清晨高血压（连续 + 晚间对照）
> 「近5个清晨均值142/88、**连续3天≥135/85**；同期晚间均值128/80、全部达标。这种『清晨高、晚间正常』是**清晨高血压**的典型形态，而清晨正是一天中心脑血管事件最高发的时段。建议：晨起1小时内、服药前、早餐前测量；把这5天清晨记录带给医生评估清晨时段的降压覆盖。注：真正的『晨峰』幅度需24小时动态血压确诊，家庭数据只能提示。」

> 例 2 · 漏服 ↔ 血压（时间相关）
> 「本周7天里打卡5天(依从率71%，略低于80%良好线)。漏服缬沙坦的两个清晨(周三、周六)当天晚间血压均值148/92，比按时服的日子(均值132/84)**高约16/8 mmHg**——时间上明显相关，说明当前药物对您是有贡献的。想想什么打断了打卡(忘记?时间安排?)，下次复诊和医生聊聊，也许有更省心的服药节奏。」

> 例 3 · 沉默型高血压（症状↔BP 错配）
> 「本周7次读数有4次≥135/85，但您**没有记录任何头晕/头痛**。这是高血压最需要警惕的状态——『没感觉≠血压正常』，多数高血压患者完全没有症状，家庭自测的价值正在于此。请继续每天记录，复诊带上数据。」

> 例 4 · 达标率回升 + 波动性提示
> 「本周达标率71%(5/7天)，比上周43%明显回升；30天均值131/82。但需注意收缩压在122–151间跳动(日间CV≈12%)——『稳定的130比波动的130更安全』，波动本身独立增加脑血管负担。波动常与漏服、高盐日、睡眠差相关，您的用药日志能帮定位。」

> 例 5 · 趋势改善 + 鼓励
> 「近7天均值126/79、达标率86%，较上月(138/86、48%)**收缩压下降约12 mmHg**——这个幅度的长期下降，循证上可显著降低卒中与心脏事件风险。同期用药依从率91%、记录连续无缺，看起来近期的管理节奏正在起作用，继续保持。」

> 例 6 · 红线/危象
> 「今天清晨读数**185/112**，达高血压危象标准(≥180/110)，伴头痛、胸闷。请**尽快就医/联系医生，不要自行加药**；保持安静坐位、休息几分钟后再测一次以确认。带上今天和近几天的记录就诊。」

## 3. LLM 上下文装配（Context Assembly）

### 3.1 预装配 vs 函数调用/工具 —— 确认：**预装配（单次）**

对 Workers 上「每日单次小结」场景，**预装配全部上下文到 prompt 一次喂入**是更优选择，不采用 function-calling 让模型回查数据。理由与证据：

- **成本/延迟/复杂度:** 单次 daily summary 数据量小(几十条读数+几十条打卡+若干档案)，一次装入 prompt 远比多轮工具往返便宜、快、易调试。Workers 无长生命周期状态，多轮调用徒增失败面。
- **减少幻觉:** 临床 LLM 文献一致指出「**信息流架构比模型选择更影响幻觉率**」，把生成任务**分解为结构化、可核对的子任务**并**用已核验的结构化数据做锚(grounding)** 显著降低编造；单次结构化上下文 + 低温度(0.1–0.3)是受推荐的稳妥范式（Asgari 2025 临床安全与幻觉评估框架 https://pmc.ncbi.nlm.nih.gov/articles/PMC12075489/ ；医学幻觉综述 grounding in structured data https://arxiv.org/html/2503.05777v2 ；属性结构化分解 https://www.microsoft.com/en-us/research/publication/attribute-structuring-improves-llm-based-evaluation-of-clinical-text-summaries/ ；Stanford 临床摘要低温度0.1降幻觉 https://web.stanford.edu/class/cs224n/final-reports/256989380.pdf ）。
- **何时才考虑工具调用:** 数据量超出上下文窗、或需动态多跳查询（本 App 都不需要）。日常小结=单次、确定、小数据→**预装配**。

### 3.2 该装什么（映射到本仓数据表）

| 维度 | 来源表 | 装配要点 |
|---|---|---|
| 读数 | `bp_records` | 取**近 14–30 天**（非现行的7条）。按时间升序。 |
| 清晨聚合 | `bp_records` (isMorning) | 晨起读数单独算均值/达标率/连续≥135/85天数；晚间对照。 |
| 依从性 | `medications`+`med_log` | 按 stage 计算近 7/14 天 PDC；列出漏服的具体日期。 |
| 档案/合并症 | `profile` | 年龄、身高/体重(BMI)、既往病史、当前长期用药名(用于判断高危合并症如糖尿病/CKD/卒中史)。 |
| 历史(可选) | `ai_summaries` | 最近一次小结内容，便于「相比上次」措辞与避免重复。 |
| 派生统计 | 计算 | 均/近期均值、达标率、近7天vs前7天趋势、CV%、症状频次、危象读数。 |

### 3.3 结构：紧凑的结构化文本（不要裸 JSON，也不要散文流水账）

- **不要**把原始 SQL 行原样塞进 prompt（现行 `ai.ts` 即如此，模型难以自行发现交叉信号）。
- **不要**纯 JSON：模型更易在「带标签的结构化文本」上组织语言；JSON 适合工具回包，不适合让模型「读懂故事」。
- **要**先输出**预计算特征区块**（已交叉好的数字），再附**原始读数快照**（供模型取用具体例子），最后给**档案/安全约束**。特征区块让模型「按证据说话」，原始行让举例真实可核。

### 3.4 具体上下文模板（直接可用，替换 `${...}`）

```
# 角色与边界
你是高血压家庭随访的健康助理。仅根据下方【数据】生成中文小结，不得编造未给出的数字或日期。
输出是「健康提醒」，非诊断；禁止任何用药/剂量增减建议；异常(≥180/110)务必提示就医。
家庭血压达标 = 收缩≤135 且 舒张≤85。清晨高血压 = 晨起读数≥135/85。

# 【档案】
年龄: ${age} | 身高/体重: ${h}/${w}(BMI ${bmi}) | 既往: ${history}
长期用药: ${meds_summary}  # 名称/剂量/时段，供判断高危背景，不出现在建议里
合并高危因素(若有): ${diabetes|ckd|stroke_history|...}

# 【周期与依从】
周期: 最近 ${days} 天  | 测量天数: ${measured_days}/${days}
用药打卡(按时段): PDC=${pdc}%  漏服日期: ${missed_dates}
依从参考线: ≥80% 良好

# 【血压特征】(已预计算)
近${days}天均值: ${avg_sys}/${avg_dia}  达标率: ${rate}%(达标${ok}/${tot})
清晨: 均值${m_avg}/${m_avg_dia} 达标率${m_rate}% 连续≥135/85 ${streak}天
晚间: 均值${e_avg}/${e_avg_dia} 达标率${e_rate}%
趋势: 近7天均值${recent7_sys}/${recent7_dia} vs 前7天${prev7_sys}/${prev7_dia} (Δ${d_sys}/${d_dia})
波动: 晨起SBP变异系数 ${cv}% (范围 ${min_s}-${max_s})
危象读数: ${crisis_count} 次 (≥180/110)

# 【症状】
周期内症状频次: ${symptom_freq}  # 如 头晕x2 头痛x1
症状-血压错配标记: ${silent|symptom_normal|aligned}  # silent=血压高但无症状

# 【原始读数】(升序，供举例; 格式: MM-DD HH:mm 晨/晚 S/D HR SpO2 [症状])
${records_compact}

# 【上次小结要点】(避免重复，供"相比"措辞)
${last_summary_short}
```

> 这套结构与 §2.1 的检查清单一一对应：模型拿到预计算的「连续3天≥135」「漏服日期清单」「CV%」「silent 标记」后，写出例 1–6 那样的句子几乎是确定的。

## 4. 推荐系统 prompt（中文）+ 输出结构

### 4.1 系统 prompt（替换 `src/server/ai.ts` 现行 `SYSTEM`）

```
你是面向高血压患者的家庭随访「健康助理」。任务：基于用户的家庭血压自测数据、用药打卡与长期档案，生成一段「近期健康小结」。

【硬约束】
1. 只能用【数据】中已给出的数字、日期、症状；禁止编造未出现的数值或记录。引用数字时与数据一致。
2. 这是健康提醒，不是诊断。禁止任何用药或剂量增减建议（如"加量/换药/停药"）。需要时可建议"带数据复诊、与医生讨论"。
3. 任一读数 ≥180/110（危象），必须在【关注建议】明确写"尽快就医，不要自行加药"。
4. 不得宣称测到了"晨峰/晨峰幅度"——家庭读数只能提示"清晨血压偏高"，真晨峰需24小时动态血压。用词区分"清晨高血压"与"晨峰"。
5. 数据不足（如<3天、无清晨读数）时，如实说明"数据不足以得出某结论"，不要硬编。
6. 语气平和、具体、像有心的高血压专科医生在做随访，不要空泛的"注意饮食/规律作息/保持心情"套话。

【分析要求——务必基于数据落地】
- 达标率：给出本周/本期达标率(%、n/N)，并与上一期对比(↑/↓)。
- 清晨维度：聚合晨起读数(均值/连续≥135/85天数)，并与晚间对照；命中清晨高血压要点出。
- 依从性↔血压：给出依从率，并尝试把漏服日期与当天/次日血压做时间关联(若数据支持)。
- 症状↔血压：指出三者之一(沉默型=血压高无症状 / 症状与血压无关 / 同时段反复相关)。
- 趋势与波动：近7天vs前7天方向；波动大时点出 CV 或范围并提示独立风险。
- 可执行下一步：给1条具体的、非用药的建议(测量时机/记录习惯/带数据复诊/与医生聊依从性障碍)。

【输出结构——严格按此 JSON 输出，键名固定，多余的不要】
返回一个 JSON 对象：
{
  "整体评估": "1–2句：本期血压总体情况 + 趋势方向 + 达标率一句话。",
  "关键发现": ["3–5条，每条一个具体发现，带数字/日期，例如『近5个清晨均值142/88，连续3天≥135/85，晚间全部达标』"],
  "关注建议": ["2–4条，每条可执行的非用药建议；若命中危象首条为就医提醒"]
}
所有文案为简体中文。发现要"到点子上"：宁可少而具体，不要多而空泛。
```

### 4.2 输出结构（固定三段 JSON）

```
{
  "整体评估": "…本期总体+趋势+达标率…",
  "关键发现": [ "…带数字/日期的具体发现…", "…", "…" ],
  "关注建议": [ "…非用药的可执行建议…", "…" ]
}
```

- 前端按这三段渲染（整体评估置顶、关键发现列表、关注建议带 icon）。现有 `ai_summaries.content` 可存**序列化 JSON 字符串**或整段文本；建议存 JSON 字符串以便结构化展示，旧文本兼容时按纯文本兜底。
- **去掉现行「150字以内」约束**：具体性需要空间；改为「关键发现3–5条、关注建议2–4条、每条一句话」的结构约束。

### 4.3 生成参数建议

- **temperature: 0.1–0.3**（临床摘要低温度降幻觉，见 §3.1 Stanford 报告）。本仓 `generateText` 调用处加 `temperature: 0.2`、`maxTokens` 放宽到 ~700。
- 模型沿用智谱 `glm-4.7`（决策 07）。若模型对「严格 JSON」不稳，用 `generateObject` + zod schema 锁 `{ 整体评估:string, 关键发现:string[], 关注建议:string[] }`（AI SDK 原生支持，比 prompt 约束更稳）。
