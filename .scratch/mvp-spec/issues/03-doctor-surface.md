# 03 — 医生端：轻量 Web 工作台

Type: grilling
Status: resolved
Blocked by: none

## Question

MVP 里医生端做到什么程度？"家庭医生随访"是核心价值，医生怎么看患者数据、怎么反馈决定闭环能否跑通。

## Answer

**轻量医生 Web 工作台。** 医生在 web 登录后：

- 看自己被关联的患者列表；
- 看单个患者的血压趋势/异常波动/达标率/近期症状与用药；
- 写随访反馈（可由 AI 辅助生成草稿，见 04），反馈置顶展示在该患者首页。

患者端是移动 web。**两端共用同一套后端 API。**

- 依据：随访是核心价值，医生必须有地方"看 + 反馈"；轻量 web 工作台是 MVP 最小可用形态，比"医生用手机小屏看 panel"更实用，也比"先不做医生端"更能验证价值。
- 解锁：AI 辅助（04）有了落点（医生工作台）；患者-医生关联流程需设计（12）；医生-患者对话形态（13）。

## Answer 更新（范围调整）

**MVP（个人自用）阶段不建医生端**——03 / 12 / 13 整体延后到多用户 + 认证阶段（用户已确认）。本决策（轻量 Web 工作台）保留为该阶段的既定方案。当前 MVP 里 AI（04）改为给**用户自己**生成"健康小结"。
