# 12 — 患者关联家庭医生的流程设计

Type: prototype
Status: resolved
Blocked by: none

## Question

B2C 模式下，患者怎么把自己的家庭医生"关联"进来，医生又怎么接受并看到该患者数据？这是随访闭环的入口，需要可交互的流程设计。

待定：
- 患者侧：如何发起关联（邀请码 / 手机号 / 扫码 / 分享链接）？
- 医生侧：如何接受关联、确认身份（避免随便认领患者）？
- 关联后：医生能看到该患者哪些数据、权限边界。
- 解除关联：任一方退出怎么处理。

交付物：用 `/prototype`（UI 分支）出一版关联流程的可点击低保真，跑通"患者邀请 → 医生接受 → 医生看到趋势 → 写反馈"。

## Answer

**延后到多用户阶段。** 个人自用 MVP 无认证、单用户，患者-医生关联流程无立足点。Out of scope for 本 effort（见 map）。
