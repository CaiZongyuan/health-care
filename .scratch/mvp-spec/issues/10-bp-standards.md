# 10 — 血压判定与"达标率"的医学标准

Type: research
Status: open
Blocked by: none

## Question

原型用了 >140/90 = 偏高、<90/60 = 偏低、正常带 90–140 / 60–90、达标率按 ≤140/90 计。这些阈值对一款**家庭随访**应用是否正确？

- 依据权威指南核实（中国高血压防治指南、家庭血压监测中国专家共识、WHO/ISH）：
  - 应该用**家庭血压**标准（如 ≥135/85 为高血压）还是**诊室**标准（140/90）？
  - "达标"的正确定义？清晨高血压（清醒前/早晨）阈值？
  - 心率、血氧（SpO₂）是否需要阈值提示，标准是什么？
- 这是医学准确性问题，直接影响产品可信度与安全。

产出：`research/10-bp-standards.md`（权威来源 + 引用）。需给出建议写入规格的阈值与达标定义。

## Research findings

详见 [`research/10-bp-standards.md`](../research/10-bp-standards.md)（交叉验证：中国高血压防治指南 2024、家庭血压监测共识、ISH 2020、ESC 2024、Mayo Clinic）。

**核心纠错**：原型用 140/90 是**诊室（office）标准，对自测应用是错的**。家庭血压的高血压切点在中文指南 / 共识 / ISH / ESC 中均为 **≥135/85 mmHg**。"偏高"触发与达标率分母都要从 140/90 降到 **135/85**，且达标 = SBP≤135 **AND** DBP≤85（不是 OR）。偏低底线 <90/60 正确，保留。

**建议写入规格的家庭血压阈值**：

| 区间 | SBP/DBP (mmHg) | 逻辑 |
|---|---|---|
| 偏低 | <90 / <60 | OR |
| 正常 | 90–119 / 60–79 | — |
| 正常高值(提醒) | 120–134 / 80–84 | 提醒，不算达标失败 |
| 偏高(家庭高血压) | ≥135 / ≥85 | OR |
| 危象(紧急) | ≥180 / ≥110 | 就医提示 UI |

- **达标率** = 家庭读数中 SBP≤135 **且** DBP≤85 的占比。
- **清晨高血压 flag** = 晨起–10:00 读数 SBP≥135 或 DBP≥85。
- **心率**：正常 60–100；<60 或 >100 提示（<50 或 >120 建议就医）。**血氧**：正常 ≥95%，**<90% = 低氧血症**（显著提示；注意 COPD 患者 88–92% 覆盖例外）。两者均为信息性提示，不计入达标率。

**仍需人类拍板（研究→决策）**：①是否采用上述区间划分（尤其"正常高值提醒"与"危象就医 UI"两个产品/安全取舍）；②**上线前必须由临床医生签字确认**（属延迟的合规工作）；③按年龄/合并症分层目标（65–79 岁、≥80 岁、糖尿病/CKD）与 COPD 血氧覆盖，留到"按用户配置"阶段，MVP 不做。
