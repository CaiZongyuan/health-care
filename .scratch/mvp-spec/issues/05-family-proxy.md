# 05 — 家属角色：MVP 仅患者本人

Type: grilling
Status: resolved
Blocked by: none

## Question

家属能不能替患者记数据、维护档案？这决定账号体系、数据归属和权限模型。

## Answer

**MVP 只做患者本人账号；家属代记/代看作为后续阶段。**

- MVP 数据归属单一（每条记录归属于一个患者账号），不引入协作者/授权模型，降低账号与权限复杂度。
- **后续阶段**（写入 Out of scope）：家属独立账号 + 患者授权代记/代看，届时定义数据归属与权限模型（谁记的、谁能看、谁能改）。
- 取舍：原型与概述都提到家属（"患者或家属维护""紧急联系人"），但用户选择 MVP 先收敛到患者本人；"降低老年门槛"在 MVP 阶段暂靠 UI 极简 + 未来家属代记/小程序触达来补。
- 解锁：账号/认证（09）无需多角色；记录数据模型不带"记录者"维度（后续再加）。
