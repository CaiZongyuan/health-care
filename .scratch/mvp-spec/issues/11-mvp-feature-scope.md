# 11 — MVP 功能范围裁剪（四个 Tab）

Type: prototype
Status: open
Blocked by: none

## Question

把原型里四个 Tab（首页记录 / 趋势 / 医生 / 我的）的全部功能，裁成 **v1（MVP 必做）/ minimal（最小可验证）/ 裁掉（后续）** 三档，确定 MVP 边界。

待裁剪项（含但不限于）：
- 记录：一天多次记录？记录编辑/删除？历史回填？心率/血氧/症状是否都进 MVP？
- 用药：用药待办勾选、提醒（依赖通知机制，见 map fog）、用药维护。
- 趋势：达标率、近 7/30 天、异常波动分析、历史明细。
- 医生：关联流程（12）、对话形态（13）、AI 草稿（04）、置顶反馈。
- 我的：长期档案（身高/体重/病史/用药/紧急联系人）哪些进 MVP。

交付物：一份 MVP 功能范围清单（可用 `/prototype` 跑一版交互骨架对照），作为产品规格的"范围"章节输入。
