# 产品规格 — 健康随访管家

> 个人自用 MVP。面向高血压、脑出血康复期及需要长期健康随访的患者。
> 一句话：**记录血压 → 看趋势/达标率 → AI 健康小结**，打开就会用。
>
> 本文档是已实现 MVP 的产品规格。规划与决策原文见 `.scratch/mvp-spec/`；开发任务见 GitHub Issues（已全部完成）。

## 1. 定位与目标用户

- **目标用户**：个人自用（开发者本人 / 家人），高血压等慢病需长期随访。
- **核心价值**：把"日常记录 + 用药管理 + 趋势分析"整合成一个简单、连续的流程；AI 把零散数据总结成可读的"近期健康小结"。
- **设计原则**：移动优先、低门槛、大号输入、少跳转。

## 2. MVP 范围（已实现）

三个主页面（底部导航：首页 / 趋势 / 我的）。

### 首页
- **AI 健康小结**：一键用智谱 GLM 基于近 7 次记录生成"近期健康小结 + 关注点"（健康提醒，非医疗诊断，含免责声明）。
- **记录血压**：高压/低压必填（大号输入），可选填心率、血氧；症状多选（头晕/恶心/呕吐/头痛/乏力/心悸/胸闷）；备注；自动判定晨起。
- **今日用药**：从长期用药清单生成当日待办，勾选打卡；可添加用药。
- **最近记录**：最近 7 条，含状态标签、症状、心率/血氧。

### 趋势
- 近 30 次概览：平均血压 + 达标率（家庭血压 ≤135/85）。
- 近 7 次 SVG 折线（高压红 / 低压蓝，正常带参考）。
- 历史明细：状态标签、清晨高血压标记、症状、心率/血氧。

### 我的
- 基本信息（年龄/身高/体重）、既往病史、紧急联系人。
- 长期用药完整管理（添加 / 启用·停用 / 删除）。

## 3. 血压标准（家庭血压，决策 10）

| 区间 | 收缩压/舒张压 (mmHg) |
|---|---|
| 偏低 | <90 / <60 |
| 正常 | 90–119 / 60–79 |
| 正常高值（提醒） | 120–134 / 80–84 |
| 偏高（家庭高血压） | ≥135 / ≥85 |
| 危象（建议就医） | ≥180 / ≥110 |

- **达标率** = 家庭读数中 收缩压≤135 且 舒张压≤85 的占比。
- **清晨高血压** = 晨起读数 ≥135/85。
- ⚠️ 上线前面向真实多用户前须临床医生签字确认（个人自用阶段为提醒）。

## 4. 技术架构

| 层 | 选型 | 说明 |
|---|---|---|
| 前端 | React 19 + TanStack Start (SSR) + Tailwind v4 + shadcn/ui (Base UI) | 移动优先，max-w-md 容器 + 底部导航 |
| 运行时/部署 | Cloudflare Workers（`@cloudflare/vite-plugin`，`nodejs_compat`） | 全球边缘（海外，国内可达，个人自用免备案） |
| 数据 | Cloudflare D1 + Drizzle ORM | `bp_records` / `medications` / `med_log` / `profile` |
| AI | Vercel AI SDK（`generateText`）+ 智谱 GLM `glm-4.7` | 经 OpenAI 兼容端点（`createOpenAI`+`.chat`） |
| 认证 | 无（个人自用，单用户直接进入） | 多用户/手机号方案留待未来 |

- **API-first**：业务经 TanStack server functions，为未来微信小程序复用后端留路。
- **时区**：统一 Asia/Shanghai（`src/lib/datetime.ts`），SSR 与客户端一致。
- **AI 安全**：输出框定为"健康提醒而非诊断"，不出具用药/剂量建议；异常值提示就医。

## 5. 暂不做（范围外 / 后续阶段）

- 多用户与认证（手机号 + 短信、`phone` 统一身份）。
- 医生端（Web 工作台 / 患者关联 / 医患对话）。
- 家属/代理账号与授权。
- 微信小程序实际构建（API-first 已为其留路）。
- 医疗器械资质 / PIPL 合规深度建设（上线前必须单独立项）。
- 通知/提醒、数据导出、硬件（蓝牙血压计）/HIS 对接。
