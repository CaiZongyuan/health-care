## 1. 产品是什么

### 一句话定位

> 面向高血压患者的个人自用移动端健康跟踪工具：**记血压 → 看趋势 → AI 智慧家庭医师观察建议**。

它解决的核心问题是：高血压患者每天量血压、吃药，但数据散落在纸条、记忆和体检报告里，无法形成连续的趋势和有意义的反馈。这个产品把**日常记录 + 用药管理 + 趋势分析 + AI 观察**整合成一个简单、连续的流程。

### 部署形态

- **Cloudflare Workers**（海外边缘，国内可达，个人自用免备案）
- 移动 web（PWA 友好），未来考虑微信小程序（后端 API-first 已为其留路）
- **不开放注册**，单用户认证（用户名/密码登录）

## 2. 核心概念词典

| 概念 | 定义 | 映射的数据表 |
|---|---|---|
| 血压记录 | 一次完整的血压测量：收缩压/舒张压（必填）、心率/血氧/症状/备注（选填）、晨起标记 | `bp_records` |
| 血压分级 | 面向已确诊高血压患者的 5 级判定：偏低/健康/尚可/偏高/危象 | `bp_records.sys` + `dia` → `getBpStatus()` |
| 达标率 | 家庭血压达标（收缩≤135 且 舒张≤85）的读数占比 | `bp_records` → `controlRate()` |
| 用药清单 | 长期服用药物的名称/剂量/作用说明/服用时段 | `medications` |
| 用药时段 | 一天中的服药节点：早晨/中午/晚上/睡前 | `medications.stages` (JSON) |
| 用药打卡 | 某药某时段某日是否已服 | `med_log` |
| AI 小结 | 由 LLM 基于近期数据生成的结构化观察建议（整体评估/关键发现/关注建议） | `ai_summaries` |
| 长期档案 | 用户的基本信息、既往病史等静态健康背景 | `profile` (key/value) |
| 补充上下文 | 用户手填的额外背景（如"虽然没打卡但一直按时服药"），供 AI 参考 | `profile.ai_extra_context` |

## 3. 功能全景

### 3.1 血压记录

> 角色：产品核心数据入口，所有分析的基础。

首页大号输入高压/低压（必填），可展开心率/血氧/症状多选/备注。支持**时间选择**（补记早上的测量）。保存时按所选时间判定是否晨起。

- **产品里的位置**：首页第一张卡，打开 App 就能看到、0 跳转开始记录。
- **对应表**：`bp_records`

---

### 3.2 血压分级

> 角色：把数字翻译成人能看懂的状态标签。

5 级（面向已确诊高血压患者）：

| 级别 | 条件 | 颜色 |
|---|---|---|
| 偏低 | <90 或 <60 | 黄 |
| 健康 | <120 且 <80 | 绿 |
| 尚可 | 120-139 / 80-89 | 蓝 |
| 偏高 | ≥140 或 ≥90 | 橙 |
| 危象·建议就医 | ≥170 或 ≥110 | 红 |

达标率（更严格的家庭血压目标 ≤135/85）单独展示，用于趋势和 AI 分析。

- **产品里的位置**：贯穿首页最近记录、趋势历史、散点着色、日历热力、AI 上下文。
- **对应表**：计算函数 `src/lib/bp.ts::getBpStatus()`

---

### 3.3 用药管理

> 角色：让吃药变成"勾一下"的待办，不再忘。

每个药设定服用时段（早晨/中午/晚上/睡前，可多选——一天 3 次选 3 个）。首页"今日用药"按时段分组显示待办，乐观更新打卡。每个药可填**作用说明**（如"降血压""抗眩晕"），显示在药名旁。

- **产品里的位置**：首页第二张卡（打卡） + 我的页（增删/启停药物）。
- **对应表**：`medications`, `med_log`

---

### 3.4 趋势分析

> 角色：把零散记录变成趋势，回答"最近怎么样"。

- **范围切换**：7次/14次/30次/7天/30天/90天
- **A · 每日均值折线**：按天聚合 sys/dia 均值，**点击点看具体数值**
- **B · 散点 + 趋势拟合**：每次读数一个点（颜色=级别），紫色虚线为线性回归拟合（↓下降/→平稳/↑上升）
- **C · 每日均值柱状**：近 14 天每天一个均值柱
- **D · 达标率环形 + 区间分布**：达标率 + 偏低/健康/尚可/偏高/危象 计数
- **E · 日历热力图**：近 28 天，每格=当日均值级别颜色
- **历史记录**：分页（20/页）+ 按状态筛选（全部/健康/尚可/偏高/危象/偏低）

- **产品里的位置**：趋势 tab。
- **对应表**：`bp_records` → `src/components/charts.tsx`

---

### 3.5 智慧家庭医师观察建议（AI）

> 角色：用 LLM 把数据变成"到点子上"的观察建议。

首页展示最近一次 AI 观察（默认折叠只看整体评估，展开看关键发现/关注建议）。生成入口在「我的」。

- **多 Provider 配置**（我的 → AI 模型配置）：智谱/DeepSeek/魔搭/OpenRouter/自定义 + **检测可用模型**（GET /models）
- **上下文装配**（`src/server/ai-context.ts`）：近 30 天血压 + 清晨聚合 + 依从 PDC + 趋势对比 + 波动 CV + 症状↔血压错配 + 档案/合并症 + 用户补充上下文 → 结构化文本喂模型
- **输出**：JSON `{整体评估, 关键发现[], 关注建议[]}`，temperature 0.2
- **定时自动生成**：Cloudflare Cron 每日 08:00（上海），按设置开关/频率（每日/每周）
- **历史 + 次数**：我的页查看历史小结和总生成次数

- **产品里的位置**：首页顶部展示 + 我的页生成/配置/历史。
- **对应表**：`ai_summaries`, `profile` (配置 key/value)

---

### 3.6 医生报告（导出）

> 角色：给医生看的一页 A4。

`/report` 页面：患者档案 + 统计 + 趋势图 + 异常清单 + 用药 + AI 小结。点「打印/导出 PDF」用浏览器另存。支持 CSV 原始数据导出（导入 Excel）。

- **产品里的位置**：趋势页右上「医生报告」入口。
- **对应表**：`bp_records`, `medications`, `profile`, `ai_summaries`

---

### 3.7 认证

> 角色：保护个人健康数据。

单用户登录（不开放注册）。用户名/密码验证 → DB-backed session token → HttpOnly cookie。路由级守卫（`__root beforeLoad`），未登录跳转 `/login`。

- **对应表**：`users`, `sessions`

## 4. 系统架构

```
┌─────────────────────────────────────────────────┐
│                 移动 Web (PWA)                    │
│  TanStack Start (SSR) + React 19 + Tailwind v4   │
│  + shadcn/ui (Base UI)                           │
├─────────────────────────────────────────────────┤
│              Cloudflare Workers                   │
│  @cloudflare/vite-plugin · nodejs_compat         │
│                                                   │
│  ┌──────────┐  ┌──────────┐  ┌───────────────┐  │
│  │ Server   │  │ Cron     │  │ AI Context    │  │
│  │ Fns      │  │ (daily)  │  │ Assembly      │  │
│  │ (RPC)    │  │          │  │               │  │
│  └────┬─────┘  └────┬─────┘  └───────┬───────┘  │
│       │              │                │          │
│  ┌────▼──────────────▼────────────────▼───────┐  │
│  │           Cloudflare D1 (SQLite)           │  │
│  │  bp_records · medications · med_log        │  │
│  │  profile · ai_summaries · users · sessions │  │
│  └───────────────────────────────────────────┘  │
│                                                   │
│  Drizzle ORM · Vercel AI SDK (generateText)      │
│  → 智谱 GLM / DeepSeek / ModelScope / OpenRouter │
└─────────────────────────────────────────────────┘
```

| 层 | 负责 | 不负责 |
|---|---|---|
| TanStack Start SSR | 路由、SSR 渲染、server fn RPC | 数据存储 |
| Server Fns | 读写 D1、AI 生成、认证 | 渲染 |
| Cron (scheduled) | 每日 AI 自动生成 | 用户交互 |
| D1 + Drizzle | 数据持久化 | 业务逻辑 |
| AI SDK + Provider | LLM 调用 | 数据存储 |

## 尾声

> 这个产品的核心价值是**连续性**——不是单次测量，而是把每天的记录变成趋势，把趋势变成 AI 能分析的上下文，把分析变成"到点子上"的观察建议。所有功能都围绕这条链路：**记录 → 趋势 → 观察 → 行动**。
